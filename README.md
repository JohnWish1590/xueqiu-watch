> 雪哨是「雪球特别关注 新帖提醒」的正式品牌名；仓库与扩展标识统一为 **xueqiu-watch**。
> 运行**不需要** WorkBuddy，不需要 Node / Python 等任何开发环境，不需要本地编译。详见下方「安装 / 快速部署」。

## 产品简介

监视雪球「特别关注」分组里的博主，他们一发新帖，就在**本机屏幕**弹出醒目的系统通知（高优先级、不自动消失、点一下直达原帖）+ **右侧磁吸常驻弹窗**（按人合并、已读消隐、支持全部一键已读）。

借用**已登录 Chrome 的免费登录态**直接调雪球半公开 API，**不需要**提取 Cookie、不需要常驻服务。可选地把新帖**推送进你的个人微信**（企业微信自建应用，见下方「企业微信推送配置」）。

> **为什么不做独立 App / 微信小程序？** 手机沙盒隔离读不了另一个 App 的数据；小程序无后台定时能力、无系统通知；独立 App iOS 保活是硬坑且需开发者账号。当前方案：电脑前用桌面弹窗（最醒目），离开电脑用企微推到手机微信（置顶会话=最接近弹窗的原生形态）。

---

## 功能特性

- **① 系统通知**：新帖时 Windows 右下角 toast，`priority:2` + `requireInteraction:true`，即便 Chrome 最小化也会在桌面出现、不自动消失。
- **② 声音提示**（可选）：用 Offscreen Document + Web Audio 播「滴-滴」声，Chrome 最小化也响；被浏览器自动播放策略拦住时自动改系统朗读。
- **③ 右侧磁吸常驻弹窗**：从屏幕右侧滑入，按博主合并未读卡片、点击打开原帖+标已读、「全部已读」一键清空；全读完自动关闭，3 分钟未读自动重弹。
- **④ 企业微信推送到微信**（可选）：同时把新帖发到你的个人微信——电脑前桌面微信跳出、离开电脑手机微信收到。
- **零配置登录检测**：借已登录的雪球标签页注入 fetch 自动取登录态，无需手动复制 Cookie。
- **一键诊断 + 运行日志**：逐环节排查登录态获取路径，出错时把日志发作者即可定位。

## 安装 / 快速部署

**一句话结论**：运行雪哨只需要一个 Chromium 内核浏览器（Chrome / Edge / Brave 等）+ 开发者模式加载（或未来的商店安装）+ 已登录雪球；**不需要 WorkBuddy，不需要 Node / Python 等任何开发环境，不需要本地编译**。

### 前置条件清单

- ✅ **Chromium 内核浏览器**：Chrome / Edge / Brave / Arc 等，版本支持 Manifest V3。
- ✅ **一个已登录 `xueqiu.com` 的浏览器标签页**，且保持打开（MV3 硬限制，否则收不到提醒）。
- ✅ **特别关注分组**：你对雪球确有「特别关注」分组（可选企微推送时才需要企业微信账号）。
- ❌ **不需要**：WorkBuddy、git、Node.js、Python、任何服务器或本地编译。

### 方式一：开发者模式加载已解压（当前唯一分发方式，推荐）

1. 下载本仓库 Release 中的 `xueqiu-watch-v1.3.0.zip`，或 `git clone` 后取仓库目录，解压。
2. 浏览器地址栏输入 `chrome://extensions`（Edge 为 `edge://extensions`）。
3. 右上角打开 **开发者模式（Developer mode）**。
4. 点 **加载已解压的扩展程序（Load unpacked）**，选择解压后的目录。
5. 浏览器右上角出现「雪哨」图标即成功。Chrome 重启自动启用，更新时点「刷新」即可。

### 方式二：Chrome 网上应用店安装（占位说明）

> ⚠️ **本期（v1.3.0）暂不上架 Chrome Web Store。** 仓库已 public，开发者模式加载已能让所有人下载使用。上架相关准备见 `STORE_GUIDE.md`（为未来上架预留）。待上架后，此处置商店搜索链接，点「添加至 Chrome」一键安装。

### 验证安装

安装后在扩展设置页点「📢 发送测试提醒」，应看到系统通知 + 右侧磁吸小窗同时出现（声音需在设置里开启）。失败请回看下方「常见问题 / 故障排查」。

## 使用说明

- **前提**：Chrome 已登录雪球（`xueqiu.com`），且你确实有「特别关注」分组。
- 扩展安装后会**立即跑一次检查**——这次只记录当前最新帖、不发通知（避免一上来弹一堆历史帖）。
- 之后按设定间隔（默认 2 分钟）轮询。发现比记录更新的帖就弹通知。
- 点通知或通知上的「打开原帖」按钮 → 在新标签页打开雪球原帖。
- 点扩展图标可看：监控人数、上次运行时间、每人抓取条数、最近新帖列表，以及「立即检查一次」。若「上次检查出错」标红，按提示排查。
- 想先验证提醒链路通不通？打开**设置页**点「📢 发送测试提醒」，会立刻弹一条测试通知 + 小窗（声音需在设置里开启）。

### 设置（点面板底部「⚙️ 设置」）

1. **检测登录**：点一下，扩展自动借已登录的雪球标签页读取登录态，显示「✅ 已登录 — 用户名：xxx」。无需 DevTools、无需手动复制 Cookie。
2. **选择要监控的博主**：点「📋 读取特别关注分组」，自动拉出你的特别关注成员列表，勾选你想监控的人即可（勾选结果自动写入监控名单）。
3. **轮询间隔**：1–60 分钟，默认 2。雪球未登录约 15 次/分钟限流，太短易触发 429。
4. **兜底博主名单**：填雪球 `user_id`（逗号或换行分隔）。当「特别关注」分组取不到时回退到此名单；留空则只依赖分组。

## 企业微信推送配置（把新帖发到你的个人微信）

扩展支持把每条新帖**额外**推送到你的个人微信，走企业微信自建应用（免费、不限量）。配置一次后，雪球新帖会同时触发：本机通知 + 声音 + 弹窗 + 微信消息。

### 前置：在企业微信后台建一个应用（个人也能建，无需公司/营业执照）

> 企业微信 ≠ 必须有公司。用你的**个人微信扫码**就能创建一个「企业」（名字随意，如「我的提醒」），创建后你就是管理员，能随意建应用、拿凭证。**这个应用纯粹是个「消息出口」，不用审核、不用上架。**

1. **进后台**：打开 [企业微信管理后台](https://work.weixin.qq.com)，用**微信扫码**登录。若提示「创建企业」，按引导建一个即可（企业名称随便填，类型选「个人 / 团队」，**无需认证**也能用推送功能）。
2. **拿 CorpID**：左侧「我的企业」→「企业信息」→ 拉到页面底部，复制 **CorpID（也叫企业 ID）**。
3. **建应用**：左侧「应用管理」→「自建」→「创建应用」→ 上传个 logo（随便选）、起名（如「雪球推送」）、「可见范围」选**你自己** → 创建。进入应用后复制 **AgentID**；点 **Secret 旁的「查看」**（会要求你用微信扫码或短信验证一次）→ 复制 **Secret**。
4. **⚠️ 关键一步：开启微信接收**：在这个应用详情页里找到「在微信中接收 / 在微信插件中接收」，把它**开启**。这一步决定消息进不进你的**个人微信**——不开的话消息只进企业微信 App，你的微信里收不到。
5. （可选）在手机微信里确认已能收到该企业微信应用的消息（微信「通讯录」→「企业微信联系人」里能看到它）。

### 在扩展里填写

打开扩展**设置页 → 企业微信推送**：

- 勾选「启用企业微信推送」
- 填入 CorpID / Secret / AgentID（就是上面拿到的三个值）
- 接收人 UserID：**直接留空即可**（= 发给应用可见范围内的所有人，也就是你自己，省去查 UserID 的麻烦）；若要指定，可在企微后台「通讯录」里查自己的 UserID，多人用 `|` 分隔
- 点「📤 测试企微推送」—— 若微信立刻收到一条测试消息，说明配置成功；否则按错误提示修正

> ⚠️ Secret 属于敏感凭证，仅保存在你本机浏览器（`chrome.storage.local`），不会上传任何服务器。

### 工作原理

新帖触发时，`background.js` 的 `pushWecom()` 会：① 用 CorpID+Secret 换 `access_token`（缓存 2 小时，提前 5 分钟复用）；② 调 `message/send` 把消息发到指定 UserID。整个过程走 `https://qyapi.weixin.qq.com`，manifest 已声明对应主机权限。

## 常见问题 / 故障排查

### 已知限制

- ✅ 本机屏幕醒目弹窗（磁吸常驻 + 全部已读）—— 已满足。
- ✅ 推到个人微信（桌面跳出 + 手机收到，只过滤特别关注、不淹没）—— 可选。在设置页「企业微信推送」配置后生效。
- ⚠️ 依赖 Chrome 常开：扩展无法在 Chrome 完全关闭后运行（这是 MV3 的硬限制）。若想要"Chrome 关了也推"，可加 GitHub Actions 定时 workflow 调企微 API（需定期更新雪球 Cookie 到 Secrets），当前版本暂不包含此功能。
- Chrome 必须保持打开且雪球已登录，否则收不到提醒。

### 故障排查：一键诊断

如果「检测登录」一直失败 / 获取不到，打开设置页**最下方**点 **「🔧 一键诊断」**：

- 它会逐条测试：① 是否有 xueqiu.com 标签页 → ② 每个标签页的 content script 是否注入并响应（次路径，未注入可忽略）→ ②b 直接用 `scripting` 注入主路径是否可用 → ③ `chrome.cookies` 是否可用 → ④ 是否手动粘贴了 Cookie → ⑤ 真实跑一次 `friendships/groups.json`（同时兼做登录探针，验证主路径端到端成功）。
- 每一步显示 ✅/❌ 及关键信息（标签页状态、返回内容预览、错误原因、修复建议）。
- 点「📋 复制报告」，把内容发给我即可精确定位问题。

> 注意：雪球原 `user/current.json` 登录校验接口已失效（返回 404），现改用 `friendships/groups.json` 作为登录探针——未登录返回 `error_code:400016`，已登录返回分组列表。

### 「有人发帖了但没提醒」怎么查

1. 先点设置页「📢 发送测试提醒」：若右下角通知 + 置顶小窗**正常弹出**，说明通知/弹窗链路没问题，问题在抓取或比对环节。
2. 点扩展图标看面板：
   - **监控博主：N 人** —— 若 N=0，说明没拉到「特别关注」分组（检查登录态、分组是否存在）。
   - **抓取: 张三(5) 李四(3)** —— 括号数字是该博主本次拉到的帖数；若某人显示 `李四:错误`，说明他时间线拉取出错（多为登录态失效）。
   - **⚠️ 上次检查出错** 标红 —— 按红字内容对症处理。
3. 仍查不出 → 设置页「🔧 一键诊断」发报告。

### 📜 运行日志（出问题时发我分析）

设置页底部有「📜 运行日志」面板。扩展在后台会**自动**记录每一步：轮询触发 / 安装 / 浏览器启动、取数走了哪条路径、每用户时间线抓取条数 / 失败原因、弹窗定位坐标、每条 ERROR / WARN 及堆栈。面板提供：**🔄 刷新 / 📋 复制 / 💾 下载 .txt / 🗑 清空**。当提醒异常又查不出原因时，直接点「📋 复制日志」或「💾 下载」把内容发我，即可精确定位。
