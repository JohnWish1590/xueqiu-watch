> 针对「雪哨（xueqiu-watch）」扩展。最后更新：2026-07-23
>
> ⚠️ **本期（v1.3.0）暂不上架 Chrome Web Store**：仓库已 public，开发者模式加载已能让所有人下载使用。本指南为未来上架预留，文案与清单均按品牌「雪哨 / xueqiu-watch」整理。

## 一、前置条件

- 一个 Google 账号
- 一次性支付 **$5** 开发者注册费（注册 Google 开发者计划）
- 访问开发者控制台：https://chrome.google.com/webstore/devconsole

## 二、准备发布包

1. 把扩展文件夹打包成 **.zip**。注意**不要**包含以下内容：
   - `.git/` 目录
   - `test_harness.js`（测试脚手架，非运行所需）
   - `STORE_GUIDE.md`、`README.md`、`PRIVACY.md`、`LICENSE`（这些是给开发者看的，不影响运行，但建议也不要打进用户包；商店后台另有描述与隐私政策字段）
   - 任何 `node_modules/`、临时文件
2. 保留必需运行文件：`manifest.json`、`background.js`、`content.js`、`popup.html/js`、`alert.html/js`、`options.html/js`、`icon16/48/128.png`。

## 四、隐私与数据披露（⚠️ 2026 年 8 月新规重点）

- **必须提供隐私政策 URL**：把 `PRIVACY.md` 托管到可公开访问的地址（推荐 GitHub Pages 或 Gist），把链接填入后台「隐私政策」字段。
- **数据使用披露（Privacy practices）**：在后台声明本扩展收集的数据类型与用途——直接对应 `PRIVACY.md` 的内容。
- **2026-08-01 起执行的新规**（已查证）：
  - 收集的用户数据**只能用于已披露的单一用途**，不得挪作他用；
  - **所有数据收集行为必须向用户披露**（无论数据类型大小）；
  - 安装后**不得偷偷变更**数据处理方式，变更须主动告知用户。
  - 本扩展的数据用途（检测特别关注新帖 + 可选企微推送）已完全对应 `PRIVACY.md`，符合要求。
  - 同时新增禁项：预测市场类扩展、绕过 AI 平台防护的扩展——本扩展均不涉及。

## 五、权限理由（Permission justification，后台逐项填写）

| 权限 | 用途 |
|------|------|
| `alarms` | 按设定间隔定时轮询特别关注时间线 |
| `notifications` | 新帖时弹出系统通知 |
| `storage` | 保存配置、未读状态、运行日志 |
| `offscreen` | 在 Chrome 最小化时也能播放提示音 |
| `windows` | 弹出提醒小窗口 |
| `tabs` | 点击通知 / 卡片时打开原帖 |
| `scripting` | 注入雪球页面内调用其 API（自动携带登录态） |
| `system.display` | 计算弹窗贴屏幕边缘的定位 |
| host: `xueqiu.com` | 读取特别关注分组与博主时间线 |
| host: `qyapi.weixin.qq.com` | 开启企微推送时调用企业微信 API |

## 六、远程代码声明

本扩展**未使用**任何远程托管的 JavaScript（所有代码均打包在扩展内，仅调用上述 API）。后台「是否使用远程代码」选 **否**。

## 七、提交审核

- 点击「提交审核」，通常 **1–2 个工作日**出结果（复杂或触发人工复核可能更久）。
- 审核结合自动 + 人工，重点关注数据收集透明度与权限必要性。
- 通过后即公开上架，用户可搜索安装。

## 八、合规提醒

- 单扩展、无重复功能、无垃圾 / 误导元数据，符合商店政策。
- 扩展功能依赖用户自己在 `xueqiu.com` 的登录态；公开发布后，未配置企微的用户仅使用本机提醒功能（属正常设计）。
- 雪球 API 为半公开接口，若雪球未来调整接口或加强风控，可能影响扩展运行——这是所有非官方客户端共有的风险。
